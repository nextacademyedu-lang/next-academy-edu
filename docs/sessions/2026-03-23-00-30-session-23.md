## Bugs Discovered

### 🔴 BUG-1 (Critical): Duplicate `About` Key in `en.json` — Raw i18n Keys on English About Page

**Symptom:** `/en/about` hero section shows raw keys like `About.heroHighlight`, `About.stat1` instead of translated text.

**Root Cause:** `src/messages/en.json` contains **two** `"About"` objects:
- Lines 281–313: original block with keys `heroHighlight`, `stat1`–`stat4`
- Lines 346–406: redesigned block with keys `heroEyebrow`, `heroStat1Value`, etc.

In JSON, duplicate top-level keys cause the second to overwrite the first. The component `about-hero.tsx` references keys from the first block, but they don't exist in the second (winning) block.

**Fix:** Merge both `About` blocks. Keep the second block's structure, but add the missing keys that `about-hero.tsx` uses:
- `heroHighlight`, `stat1`, `stat2`, `stat3`, `stat4`

**Files:** `src/messages/en.json`, `src/components/sections/about-hero.tsx`

---

### 🔴 BUG-2 (Critical): Booking Flow Returns 401 for Authenticated Users

**Symptom:** After successful login, clicking "Book Now" on a program triggers `POST /api/bookings/create` which returns `401 Unauthorized`.

**Root Cause Analysis:**
- `book-round-button.tsx` sends request with `credentials: 'include'` ✅
- `server-auth.ts` checks 3 token strategies: `payload-token` cookie → JWT header → Bearer token
- All 3 fail, meaning the `payload-token` cookie is either not set at login, not sent on API calls, or rejected by domain/path/SameSite mismatch

**Likely Cause:** Cookie attributes (`Domain`, `SameSite`, `Secure`, `Path`) are misconfigured between the login response and the API request origin.

**Investigation Steps:**
1. Check browser DevTools → Application → Cookies for `payload-token` presence after login
2. Check the cookie's `Domain` and `Path` attributes
3. Verify `NEXT_PUBLIC_SERVER_URL` matches the actual domain serving the API

**Files:** `src/lib/server-auth.ts`, `src/components/checkout/book-round-button.tsx`

---

### 🟡 BUG-3 (Medium): Terms Page Shows Popup with Test Data

**Symptom:** `/en/terms` displays a popup overlay containing clearly fake test data: "asd", "f2dsa", "dcas".

**Root Cause:** A test popup was likely created in the Payload CMS admin panel and left in active/published state. The popup targets the Terms page or all pages.

**Fix:** Go to Payload Admin → Popups → find and either delete or set to draft status.

### 🟢 BUG-4 (Minor): Test Program "sad" in Program Listing

**Symptom:** A program with slug "sad" appears in the programs grid with no instructor and no description.
