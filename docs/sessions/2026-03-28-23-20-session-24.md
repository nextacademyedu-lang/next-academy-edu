## Bug Fixed

### Issue
Deleting users in production returned `400` with:

`The following path cannot be queried: user`

### Root Cause
In `Users.beforeDelete`, cleanup attempted to query `verification-codes` using field `user`, but that collection uses `email`.

### Fix
Updated `src/collections/Users.ts`:

- Fetch target user first in `beforeDelete`.
- Cleanup `verification-codes` by `email` instead of `user`.
- Made sub-cleanups resilient (log warnings, do not block primary user delete).
- Added extra cleanup path for `installment-requests.reviewedBy`.
